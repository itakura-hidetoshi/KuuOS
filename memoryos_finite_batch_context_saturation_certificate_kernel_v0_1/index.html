    MemoryOS v0.92 — Finite Batch Counterexample Context Saturation Certificate Kernel¶

    Scope¶

    MemoryOS v0.92 advances the v0.91 one-query counterexample repair into a finite batch refinement and saturation theorem.

    For a finite sensor index type (I), v0.91 associates each premise support (S) with the canonical counterexample context

    [ C_S=\operatorname{cl}_r(S), ]

    whenever this closure is proper, meaning (C_S\ne I).

    For a finite premise batch (B), define the distinct proper closure image

    [ \mathcal G_r(B) = \left{ \operatorname{cl}_r(S) \mid S\in B,\ \operatorname{cl}_r(S)\ne I \right}. ]

    The finite batch refinement of a selected context family (\mathcal A) is

    [ \operatorname{BatchRefine}_r(\mathcal A,B) = \mathcal A\cup\mathcal G_r(B). ]

    The construction is exact, finite, deterministic, read-only, and advisory.

    Literature alignment¶

    The immediate 2026 trigger remains arXiv:2607.01773, where Formal Concept Analysis drives a multi-round verification loop: candidate implications are accepted or receive inspectable counterexamples, and the formal context is updated across rounds.

    The exact finite layer is additionally aligned with:

    • arXiv:2205.15714 — exploration with multiple partial and potentially contradicting experts;
    • arXiv:1202.4824 — general attribute exploration as knowledge completion;
    • arXiv:1503.09025 — exact learning of definite Horn theories from closure and equivalence queries;
    • arXiv:1810.08684 — one-set updates of finite closure systems and implicational bases;
    • arXiv:2404.12229 — the distinction between minimal and direct bases;
    • arXiv:1110.5805 — ordered direct bases for finite closure systems.

    MemoryOS imports only the finite closure/counterexample pattern. It does not import an empirical oracle, retrieval authority, optimal query scheduler, or minimum-basis claim.

    Canonical proper closure image¶

    Lean proves exact membership:

    [ C\in\mathcal G_r(B) \iff \left( \exists S\in B,\ \operatorname{cl}_r(S)=C \right) \land C\ne I. ]

    Every generated context is closed and proper. If the source family (\mathcal A) already consists of proper closed contexts, then so does (\operatorname{BatchRefine}_r(\mathcal A,B)).

    Because Finset.image and Finset union remove duplicates, premise multiplicity and enumeration order have no semantic effect.

    Exactness for every premise in the batch¶

    For every (S\in B) and every sensor (i),

    [ \boxed{ i\in\operatorname{cl}r(S) \iff S\models{\operatorname{BatchRefine}_r(\mathcal A,B)} i } ]

    provided (\mathcal A) contains only proper closed contexts.

    Thus one finite insertion repairs all singleton queries whose premises occur in the batch, not only one selected conclusion.

    If (\operatorname{cl}_r(S)=I), every sensor consequence is already true. Otherwise the inserted context (\operatorname{cl}_r(S)) separates every sensor outside the exact closure.

    Algebra of finite batches¶

    Idempotence¶

    [ \operatorname{BatchRefine}_r( \operatorname{BatchRefine}_r(\mathcal A,B),B ) = \operatorname{BatchRefine}_r(\mathcal A,B). ]

    Union composition¶

    [ \boxed{ \operatorname{BatchRefine}_r(\mathcal A,B_1\cup B_2) = \operatorname{BatchRefine}_r( \operatorname{BatchRefine}_r(\mathcal A,B_1),B_2 ) } ]

    Commutativity of sequential batches¶

    [ \operatorname{BatchRefine}_r( \operatorname{BatchRefine}_r(\mathcal A,B_1),B_2 ) = \operatorname{BatchRefine}_r( \operatorname{BatchRefine}_r(\mathcal A,B_2),B_1 ). ]

    Premise-family monotonicity¶

    If (B_1\subseteq B_2), then

    [ \operatorname{BatchRefine}_r(\mathcal A,B_1) \subseteq \operatorname{BatchRefine}_r(\mathcal A,B_2). ]

    Closure-image invariance¶

    If (\mathcal G_r(B_1)=\mathcal G_r(B_2)), then

    [ \operatorname{BatchRefine}_r(\mathcal A,B_1) = \operatorname{BatchRefine}_r(\mathcal A,B_2). ]

    This is the exact redundancy law: different premise batches that generate the same distinct closure contexts are observationally identical.

    Finite saturation¶

    Let (\mathcal P(I)) denote the finite family of all sensor supports. Lean proves

    [ \boxed{ \operatorname{BatchRefine}_r( \varnothing,\mathcal P(I) ) = \mathcal C_r^\circ } ]

    where (\mathcal C_r^\circ) is the family of all proper closed sensor supports from v0.91.

    Every proper closed context (C) is generated by the premise (S=C), because (\operatorname{cl}_r(C)=C).

    Consequently the saturated family is complete for every singleton and finite-support consequence:

    [ i\in\operatorname{cl}r(S) \iff S\models{\mathcal C_r^\circ} i, ]

    [ T\subseteq\operatorname{cl}r(S) \iff S\models{\mathcal C_r^\circ} T. ]

    This is a finite convergence theorem. It is not a minimum-context theorem and does not claim that every support must be queried in an efficient exploration procedure.

    Root independence¶

    Closure is root independent, so the canonical closure image and every finite batch refinement are root independent.
