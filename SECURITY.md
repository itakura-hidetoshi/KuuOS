# Security Policy

## Scope

KuuOS is currently a public governance and validation repository.

This repository is not currently presented as a production deployment system.

## Reporting

If you identify:

- validator bypasses
- provenance bypasses
- silent authority escalation paths
- unsafe execution pathways
- audit-chain inconsistencies
- credential exposure
- unsafe workflow behavior

please report them privately to the repository maintainer.

## Current Security Orientation

KuuOS emphasizes:

- fail-closed behavior
- provenance visibility
- append-only governance records
- explicit uncertainty
- rollback visibility
- non-authority preservation

## Out of Scope

The following are currently outside the public repository scope:

- private research kernels
- unpublished implementation layers
- private datasets
- operational credentials
- clinical systems
- institutional infrastructure

## Validation Boundaries

Passing repository validation does not imply production safety.

Validation currently focuses on structural governance consistency.

## Responsible Disclosure

Please avoid publicly disclosing vulnerabilities that could:

- bypass governance boundaries
- create unsafe execution escalation
- compromise provenance tracking
- expose credentials or private data
