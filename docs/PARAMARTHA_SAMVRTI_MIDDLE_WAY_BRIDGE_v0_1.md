# Paramartha–Samvrti–Middle Way Bridge v0.1

## 勝義諦・世俗諦・中道ブリッジ

This document defines how KuuOS / 空OS operationalizes 二諦 and 中道.

## 1. 勝義諦 / paramartha-satya

In KuuOS, 勝義諦 is not a highest substance, not an absolute object, and not a final metaphysical entity.

It is the non-reifying truth of emptiness:

- no independent self-nature,
- no isolated final authority,
- no self-grounded output,
- no model surface that becomes reality itself,
- no proof that becomes unrestricted execution authority.

Operational role:

```text
paramartha_surface:
  function: block_reification
  denies:
    - absolute_self_authority
    - final_identity_fixation
    - hidden_totalization
    - proof_as_unbounded_execution
    - model_output_as_world_itself
```

## 2. 世俗諦 / samvrti-satya

世俗諦 is the conventional operating surface.

It includes:

- language,
- models,
- specifications,
- proofs,
- plans,
- metrics,
- governance rules,
- clinical and engineering workflows,
- world-model descriptions.

世俗諦 is not false merely because it is conventional. It is valid when its conditions, scope, evidence, lineage, and constraints are visible.

Operational role:

```text
samvrti_surface:
  function: enable_responsible_operation
  requires:
    - scope
    - conditions
    - lineage
    - evidence_or_support
    - governance_boundary
    - visible_residuals
```

## 3. 中道 / Middle Way

中道 is the bridge that prevents collapse.

It prevents two failures:

```text
reification_failure:
  error: treating 世俗諦 surfaces as self-grounded absolutes

nihilistic_failure:
  error: using 勝義諦 to erase responsibility, action, care, or repair
```

Middle Way preserves both:

- the non-reifying force of 勝義諦,
- the responsible operational surface of 世俗諦.

## 4. KuuOS Runtime Bridge Rule

A KuuOS candidate is valid only when it can pass through this bridge:

```text
candidate
  -> emptiness_check
  -> dependent_origination_trace
  -> two_truths_separation
  -> middle_way_bridge
  -> governed_conventional_operation
```

If the candidate cannot distinguish 勝義諦 from 世俗諦, it must not be promoted.

If the candidate rejects 世俗諦 operation in the name of emptiness, it must not be promoted.

If the candidate absolutizes 世俗諦 output, it must not be promoted.

## 5. Decision Consequences

```text
if reification_failure:
  status: HOLD or FAIL
  action: de-reify, rescope, add conditions

if nihilistic_failure:
  status: HOLD or REVIEW
  action: restore responsibility, care, and conventional support

if two_truths_confused:
  status: HOLD
  action: separate paramartha surface and samvrti surface

if middle_way_bridge_passes:
  status: eligible_for_governed_review
```

## 6. Relation to KuuOS Modules

- MemoryOS preserves lineage without making memory absolute.
- BeliefOS licenses belief without turning belief into reality itself.
- PlanOS creates plans without making plans self-authorizing.
- DecisionOS issues advisory licenses without direct execution authority.
- ReflectionOS repairs and learns without erasing prior traces.
- RuntimeGovernance prevents collapse into either absolutism or abandonment.
