# Formal Terminology Glossary v0.1

## Purpose

This glossary provides stable public terminology for KuuOS reviewers.

## Core Terms

| Term | Meaning in KuuOS |
|---|---|
| Candidate | A generated or proposed surface that has not acquired authority |
| Authority | A scoped permission or status granted by explicit governance process |
| Boundary | A declared separation between layers, roles, or authority classes |
| Non-authority | A surface that may inform but cannot decide or execute by itself |
| Provenance | Traceable lineage of origin, transformation, and review |
| Receipt | A record of validation, decision, handoff, or audit event |
| Fail-closed | Default refusal to proceed when required conditions are missing |
| Hold | Legitimate non-executing state preserving unresolved conditions |
| Abstain | Legitimate refusal to claim or act under insufficient conditions |
| Rollback | Return to an earlier safe or reviewable state |
| Reobserve | Request for additional observation before claim or action |

## Interpretation

This glossary is a public terminology surface.
