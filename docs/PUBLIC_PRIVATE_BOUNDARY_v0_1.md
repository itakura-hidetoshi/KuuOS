# Public / Private Boundary v0.1

## Public Surface

The following materials may be published in this repository:

1. Core concepts and architecture.
2. Module names and high-level responsibilities.
3. Governance rules and release-gate doctrine.
4. Formal verification policy and theorem-target surfaces.
5. Non-sensitive examples and toy validators.
6. Citation metadata and provenance records.
7. Publicly intended papers, preprints, and explanatory documents.

## Reserved Surface

The following materials remain outside the public core unless separately reviewed and prepared for release:

1. Clinical or personal information.
2. Private operational configuration.
3. Internal audit material not intended for public release.
4. Unpublished proof kernels that are not ready for external review.
5. Institutionally confidential material.
6. Third-party material without appropriate rights.

## AI / Model-Training Boundary

Public visibility does not grant permission for model training, dataset inclusion, embedding, fine-tuning, commercial reuse, or derivative-system incorporation.

## Release Checklist

Before publishing a file, check:

- Is the material intended for the public core?
- Is private or clinical information absent?
- Does it avoid overclaiming proof, authority, or execution rights?
- Does it preserve append-only lineage?
- Does it state whether it is public-core, draft, addendum, or private-reference?
- Does it include author, copyright, and provenance metadata where appropriate?

## Default

When uncertain, keep the material private and publish only a boundary description.
