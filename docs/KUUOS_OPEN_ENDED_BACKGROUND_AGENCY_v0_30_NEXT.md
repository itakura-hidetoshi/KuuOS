# Next materialization after v0.30

v0.30 establishes the non-contraction constitution and transition kernel. The next implementation layer should materialize open-ended operation without redefining open-endedness as merely repeated fixed bounded cycles.

Priority order:

1. endogenous mission-candidate formation from unresolved WORLD evidence;
2. autonomous observation-seeking portfolio generation;
3. durable background event intake and wake-up;
4. WORLD-fragment creation, comparison and revision;
5. validated tool and subsystem creation;
6. recoverable long-horizon memory reinterpretation;
7. dynamic multi-agent formation and retirement;
8. resource discovery, composition and negotiation;
9. production service, HA and deployment realization.

Each later layer must preserve:

```text
local control != global constitution
candidate != authority
implementation limit != permanent possibility closure
```
