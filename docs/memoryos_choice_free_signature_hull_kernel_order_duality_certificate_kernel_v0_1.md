## Choice-free envelope and closed hull

For a finite antichain signature `A`, define its envelope by

```text
Env(A) = union of every support M in A.
```

Define its closed hull by

```text
Hull_r(A) = cl_r(Env(A)).
```

For a proper closed context `C`, v0.94 supplies the complete signature

```text
Sig_r(C) = { M | M is closure-minimal and cl_r(M) = C }.
```

MemoryOS v0.95 proves

```text

```

The envelope itself need not equal `C`. For example, a context may be generated by the empty support. Closure of the complete envelope is the exact reconstruction invariant.

## Signature-hull order

For antichain signatures `A` and `B`, define

```text
A <=_Hull B  iff  Hull_r(A) subset Hull_r(B).
```

For proper closed contexts `C,D`, the Lean layer proves

```text
Sig_r(C) <=_Hull Sig_r(D)
  iff C subset D.
```

Closed-context inclusion is exactly reverse invisible-kernel inclusion:

```text
C subset D
  iff K_r^D <= K_r^C.
```

Therefore

```text
Sig_r(C) <=_Hull Sig_r(D)
  iff K_r^D <= K_r^C.
```

For proper support closure classes `S,T`,

```text
ClassSig_r(S) <=_Hull ClassSig_r(T)
  iff cl_r(S) subset cl_r(T)
  iff K_r^T <= K_r^S.
```

## Antisymmetry

If two proper closed-context signatures are mutually below each other in hull order, their represented contexts are equal and the signatures themselves are equal.

```text
Sig_r(C) <=_Hull Sig_r(D)
and Sig_r(D) <=_Hull Sig_r(C)
implies Sig_r(C) = Sig_r(D).
```
