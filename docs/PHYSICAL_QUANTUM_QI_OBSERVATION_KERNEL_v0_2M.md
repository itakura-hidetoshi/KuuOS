# Physical Quantum Qi Observation Kernel v0.2M

Status: CANDIDATE_RUNTIME_SURFACE

Date: 2026-05-19

Repository: itakura-hidetoshi/KuuOS

Runtime kind: observation sequence to non-authoritative Qi state candidate kernel

## Purpose

`Physical Quantum Qi Observation Kernel v0.2M` adds the minimal executable projection from an observation event sequence into a bounded `QiStateCandidate`.

The kernel extends the established Physical Quantum Qi line without replacing v0.2F Process Tensor, v0.2G conventional autonomy, v0.2J Bensho treatment route candidate, v0.2K clinician handoff boundary, or v0.2L clinical red flag consultation governor.

## Canonical interpretation

- Process Tensor remains the multi-time non-Markov temporal structure.
- Qi is a history-bearing relational/action/recoverability flow on that temporal structure.
- Observation is not free.
- Observation backaction must remain visible.
- Source trace must remain visible.
- Process history must not be replaced by a current snapshot.
- Qi state projection is candidate only.
- Qi is operator-valued flow, not substantial essence.
- Clinical consultation boundary is preserved.

## Minimal projection

The runtime projection is:

```text
observation event sequence
  -> process-tensor context check
  -> non-Markov memory/resource metrics
  -> bounded QiStateCandidate
  -> OS candidate surfaces
  -> DecisionOS safety-evaluable candidate only
```

The runtime does not simulate quantum mechanics. It performs bounded runtime projection and governance validation.
