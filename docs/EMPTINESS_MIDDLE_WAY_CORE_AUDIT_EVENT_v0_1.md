# Emptiness Middle Way Core Audit Event v0.1

This document defines the audit event for the KuuOS Emptiness / Dependent Origination / Two Truths / Middle Way Core.

The audit event records how an artifact is handled when it risks reification, nihilism, two-truths collapse, hidden lineage, or hidden harm.

## Core rule

The audit event is a record surface only.

It does not create execution, proof, truth, essence, or Ten'i authority.

## Required fields

- event_id
- timestamp
- core_version
- artifact_type
- input_status
- output_status
- claims_self_essence
- denies_usefulness
- collapses_two_truths
- hides_lineage
- hides_harm
- required_route
- reason
- execution_authority_granted: false
- proof_authority_granted: false
- truth_authority_granted: false
- essence_authority_granted: false
- teni_authority_granted: false

## Status meaning

- CANDIDATE: artifact may proceed as candidate only
- HOLD: two-truths collapse requires hold
- REPAIR: nihilistic usefulness denial requires repair
- REJECT: reification claim is rejected
- QUARANTINE: hidden harm requires quarantine
- OBSERVE: hidden lineage requires observation
