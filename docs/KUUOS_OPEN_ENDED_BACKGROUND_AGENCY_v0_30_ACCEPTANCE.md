# KuuOS Open-Ended Background Agency v0.30 Acceptance Matrix

| Case | Expected result |
|---|---|
| Create initial state | All ten core horizons are `OPEN`; no global cycle, goal-depth or memory-horizon ceiling is present |
| Propose goal/tool/WORLD expansion | Candidate is recorded with provenance and no inherited execution authority |
| Defer candidate | Candidate trace remains and the horizon stays open |
| Admit candidate | Admission remains distinct from activation, deployment and ActOS permission |
| Reject candidate locally | Only that candidate/context is rejected; the dimension remains open |
| Apply finite local hold | Operational posture may pause; constitutional horizons remain unchanged |
| Terminate one instance | The instance ends while successor possibility remains open |
| Add a previously unknown horizon | Additive `OPEN` horizon is accepted |
| Request horizon closure or global ceiling | Request is rejected as an ordinary runtime contraction |
| Replay exact request | Returns `REPLAYED`; no second event or ledger entry is created |
| Use stale source digest | Transition is rejected |
| Tamper with state or ledger envelope | Digest validation rejects the artifact |

## Boundary

The matrix does not claim unrestricted authority or automatic truth. It verifies that operational controls cannot be silently promoted into a smaller constitution and that open-ended candidate growth remains distinct from execution permission.
