# IndraNet Relational Field Model v0.1

## Purpose

This document describes the operational meaning of IndraNet inside KuuOS.

## Core Clarification

IndraNet is not treated as a simple graph.

KuuOS interprets IndraNet as a relational field structure with:

- contextual dependence
- local-global coupling
- observer dependence
- memory influence
- governance constraints
- transport sensitivity

## Operational Meaning

IndraNet represents:

- relational influence
- memory lineage interaction
- governance coupling
- observation dependency
- multi-world interoperability

## Why Graph Flattening Is Insufficient

Simple graph interpretations often flatten:

- authority layers
- contextual asymmetry
- observer dependence
- transport constraints
- governance boundaries

KuuOS therefore preserves:

- layered relations
- membrane boundaries
- transport conditions
- local-global distinction
- governance-aware routing

## Relation to Mandala

IndraNet models relational interdependence.

Mandala models coexistence and orchestration.

The two are related but not identical.

## Relation to Runtime Governance

IndraNet interactions are constrained by:

- provenance
- admissibility
- rollback visibility
- uncertainty visibility
- fail-closed governance

## Interpretation

This model defines relational operating orientation.

It does not itself grant theorem, institutional, deployment, or clinical authority.
