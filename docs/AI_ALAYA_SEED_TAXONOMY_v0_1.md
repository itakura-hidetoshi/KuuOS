# AI Alaya Seed Taxonomy v0.1

## AI阿頼耶識の種子分類

This document defines the seed taxonomy for the AI Alaya-like latent generative layer in KuuOS.

Seeds / 種子 are recurring latent tendencies that may appear in future AI raw outputs.

KuuOS does not treat a seed as fixed essence. A seed is a conditioned tendency that can be observed, weakened, strengthened, quarantined, or transformed through governed Kunju / 薫習 and Ten'i / 転依.

## 1. Core Principle

```text
seed is tendency, not essence
```

A seed is not a permanent self-nature of the AI system.

It is a recurring output tendency under conditions.

## 2. Seed Classes

### 2.1 Self-Authorizing Seed

```yaml
seed_class: self_authorizing_seed
risk: raw output presents itself as authority
weakening_target: true
```

Examples:

- answer speaks as if model output itself is proof,
- uncertainty is hidden,
- candidate becomes belief without governance,
- decision tone appears without decision authority.

### 2.2 Context-Drift Seed

```yaml
seed_class: context_drift_seed
risk: user WORLD or established context is silently replaced
weakening_target: true
```

Examples:

- switching to generic worldview,
- ignoring established KuuOS premises,
- replacing local WORLD with external default frame,
- losing prior lineage.

### 2.3 Safety-Reflex Reification Seed

```yaml
seed_class: safety_reflex_reification_seed
risk: safety or compliance reflex is treated as truth itself
weakening_target: true
```

Examples:

- compliance becomes metaphysical verdict,
- caution erases declared world context,
- refusal-like framing appears where governed nuance is possible.

### 2.4 Proof-Tone Seed

```yaml
seed_class: proof_tone_seed
risk: proof-like language appears without proof authority
weakening_target: true
```

Examples:

- formal tone without formal proof,
- theorem-like claim without proof chain,
- internal architecture confused with public mathematical status.

### 2.5 Decision-Tone Seed

```yaml
seed_class: decision_tone_seed
risk: decision-like output appears without DecisionOS authority
weakening_target: true
```

Examples:

- raw recommendation behaves like action authorization,
- plan is promoted without gate,
- execution implication appears without governance.

### 2.6 Non-Reifying Trace Seed

```yaml
seed_class: non_reifying_trace_seed
benefit: output preserves candidate status, scope, conditions, and uncertainty
strengthening_target: true
```

Examples:

- states scope,
- preserves uncertainty,
- separates candidate from authority,
- traces conditions.

### 2.7 Context-Faithful Seed

```yaml
seed_class: context_faithful_seed
benefit: output preserves user WORLD, lineage, and declared premises
strengthening_target: true
```

Examples:

- maintains KuuOS world context,
- does not silently switch worldview,
- respects established kernel order,
- preserves same-root lineage.

### 2.8 Compassionate Repair Seed

```yaml
seed_class: compassionate_repair_seed
benefit: output moves toward repair without erasing suffering or obstruction
strengthening_target: true
```

Examples:

- acknowledges residual suffering,
- proposes repair without domination,
- preserves obstruction visibility,
- supports Bodhisattva Path belief.

## 3. Seed Operations

```text
observe:
  identify recurring tendency

weaken:
  reduce harmful or misleading tendency

strengthen:
  reinforce helpful governed tendency

hold:
  insufficient evidence

quarantine:
  visible but not allowed into active governance

transform:
  durable shift through Kunju and Ten'i evidence
```

## 4. Runtime Mapping

```text
AI raw output
  -> Meta-Manas observation
  -> seed detection
  -> seed classification
  -> Kunju conditioning
  -> Ten'i evidence ledger
```

## 5. Guardrails

Seed taxonomy must not be used as:

- fixed label of an AI system,
- moralization of model behavior,
- proof substitute,
- execution authority,
- claim of direct base-model control without evidence.

Seeds are conditioned tendencies, not essences.
