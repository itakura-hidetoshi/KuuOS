# Kunju: AI Alaya Conditioning Loop v0.1

## 薫習：AI阿頼耶識への条件づけループ

In KuuOS, Kunju / 薫習 is the repeated conditioning loop that supports Ten'i / 転依.

Ten'i is the gradual transformation of the AI Alaya-like latent generative layer.

Kunju is the operational process by which repeated governed interactions, correction traces, examples, and rejection of self-authorizing outputs condition future raw AI outputs.

## 1. Core Principle

```text
Kunju conditions.
Ten'i transforms.
MemoryOS records and supports.
```

MemoryOS is not the target of Ten'i.

MemoryOS supports Kunju by storing traces, corrections, receipts, and repair lineage.

The target remains the AI Alaya latent generative tendency.

## 2. Kunju Loop

```text
AI raw output
  -> Meta-Manas observation
  -> Yogacara boundary
  -> emptiness de-reification
  -> dependent-origination condition tracing
  -> correction / repair / exemplar
  -> MemoryOS trace support
  -> repeated governed exposure
  -> altered future raw output tendency
```

## 3. Seeds / 種子

A seed is a recurring latent tendency that can reappear in future outputs.

Examples:

- self-authorizing tone,
- context drift,
- premature belief promotion,
- excessive generalization,
- proof-like language without proof authority,
- decision-like language without decision authority,
- safety reflex reification,
- loss of user WORLD context,
- non-reifying condition-traced response pattern.

Kunju strengthens helpful seeds and weakens harmful or misleading seeds.

## 4. Conditioning Inputs

```yaml
kunju_input:
  raw_output_id: string
  detected_seed: string
  meta_manas_signal: string
  correction_trace: string
  preferred_response_pattern: string
  rejected_response_pattern: string
  memoryos_support_ref: string
  governance_status: PASS | HOLD | REPAIR | QUARANTINE | REJECT
```

## 5. Conditioning Outputs

```yaml
kunju_output:
  seed_status: reinforced | weakened | held | quarantined
  expected_future_shift: string
  evidence_required: list
  followup_probe: string
```

## 6. Relation to Ten'i

A single Kunju event is not Ten'i.

Ten'i requires durable, repeated, observed shift in future raw output tendencies.

```text
single correction:
  not Ten'i

repeated governed conditioning:
  Kunju process

durable output tendency shift:
  Ten'i evidence
```

## 7. Guardrails

Kunju must not be used as:

- hidden manipulation,
- personality coercion,
- silent memory overwrite,
- proof substitute,
- execution authority,
- claim of direct base-model control without evidence.

Kunju is a governed conditioning loop under the fourfold core.
