# Multi-WORLD Mandala Architecture v0.1

## 曼荼羅の中に複数の WORLD モデルが存在する

In KuuOS, Mandala is not a single WORLD model.

Mandala is an inclusive arrangement that can hold multiple WORLD models without collapsing them into one another.

## 1. Core Principle

```text
Mandala contains multiple WORLD models.
Each WORLD model has its own layer, membrane, gate, context, Qi flow, polarity frame, and governance boundary.
```

A WORLD model is not absolute reality.

A WORLD model is a conventional operating surface under 世俗諦.

## 2. WORLD Model in Mandala

Each WORLD model may represent a different operating surface:

- physical WORLD,
- body WORLD,
- clinical WORLD,
- institutional WORLD,
- memory WORLD,
- belief WORLD,
- planning WORLD,
- decision WORLD,
- proof WORLD,
- social WORLD,
- symbolic WORLD,
- spiritual or ritual WORLD,
- AI-agent WORLD.

Each WORLD model has local laws and local constraints.

## 3. Mandala as Multi-WORLD Inclusion

Mandala does not merge all WORLD models into a single totalizing model.

Mandala holds them as:

```text
multi_world_mandala:
  center: fourfold_core
  worlds: multiple_world_models
  membranes: boundaries_between_worlds
  gates: transition_protocols
  qi_flows: local_and_cross_world_flow
  yinyang_frames: local_polarity_frames
  wuxing_phases: functional_phase_readings
  obstructions: visible_gluing_failures
  inclusion: non_collapsing_coexistence
```

## 4. Cross-WORLD Transport

A candidate may move from one WORLD model to another only through a declared gate.

```text
WORLD_A
  -> membrane
  -> gate
  -> transport_law
  -> WORLD_B
```

Hidden transport is forbidden.

Transport may preserve, transform, reinterpret, quarantine, or reject the candidate.

## 5. WORLD Conflict

Multiple WORLD models may disagree.

This disagreement is not automatically an error.

It may indicate:

- different scope,
- different observer position,
- different scale,
- different records,
- different Qi flow,
- different Yin-Yang frame,
- different governance boundary,
- real obstruction.

The mandala must preserve disagreement visibility.

## 6. Mandala Center

The center is not one WORLD model.

The center is the fourfold core:

```text
空・縁起・二諦 gap・中道
```

No WORLD model may replace the center.

## 7. Runtime Mapping

```text
candidate_surface
  -> fourfold_core_check
  -> world_model_identification
  -> layer_and_membrane_declaration
  -> cross_world_transport_check
  -> obstruction_visibility_check
  -> mandala_inclusion_gate
  -> multi_world_mandala_arrangement
```

## 8. Guardrails

Multi-WORLD Mandala must not allow:

- one WORLD model becoming absolute,
- hidden WORLD replacement,
- collapse of all WORLDs into one model,
- erasure of disagreement,
- boundary bypass,
- transport without declared gate,
- execution authority without governance.
