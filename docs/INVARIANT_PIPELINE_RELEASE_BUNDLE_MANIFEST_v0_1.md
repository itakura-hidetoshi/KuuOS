# Invariant Pipeline Release Bundle Manifest v0.1

## 空OS Invariant Pipeline Release Bundle Manifest

This document defines the release bundle manifest for the KuuOS invariant governance pipeline.

The bundle covers:

- Formal Invariant Spine
- Super-Relativity Invariant Bridge
- Invariant Preservation Matrix
- Invariant Gate Runtime
- Invariant Governance Pipeline
- Audit Event
- Audit Hash-Chain Ledger
- WORM Export Receipt
- Public navigation / release / PR addenda

## 1. Core Principle

```text
Release bundle integrity is not authority.
```

The release bundle manifest proves only that a public file set has a stable integrity digest. It does not prove truth, Ten'i, clinical authority, proof authority, or execution authority.

## 2. Generated Manifest

The generated manifest path is:

```text
specs/invariant_pipeline_release_bundle_manifest_v0_1.generated.json
```

It is generated by:

```bash
python3 scripts/build_invariant_pipeline_release_bundle_manifest_v0_1.py
```

and validated by:

```bash
python3 scripts/validate_invariant_pipeline_release_bundle_manifest_v0_1.py
```

## 3. Bundle Root Hash

The bundle root hash is computed from ordered path + sha256 pairs:

```text
bundle_root_hash = sha256(concat(sorted(path + sha256)))
```

## 4. Required Fixed Points

```text
release_bundle_manifest_is_integrity_surface_not_authority
bundle_root_hash_proves_file_set_integrity_not_truth
bundle_manifest_does_not_prove_teni_occurrence
bundle_manifest_does_not_grant_execution_authority
bundle_manifest_does_not_replace_worm_receipt
```

## 5. Guardrails

The bundle manifest must not be used as:

- execution authority,
- proof authority,
- clinical authority,
- truth proof,
- Ten'i proof,
- replacement for evidence,
- replacement for audit hash-chain,
- replacement for WORM export receipt.
