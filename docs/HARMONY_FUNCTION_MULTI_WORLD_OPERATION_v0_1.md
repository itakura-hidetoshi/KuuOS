# Harmony Function for Multi-WORLD Operation v0.1

## 複数WORLDモデル運用のための和の関数

In KuuOS, the Harmony Function / 和の関数 exists to operate multiple WORLD models inside the Mandala.

Harmony does not mean forcing all WORLD models into sameness.

Harmony means non-collapsing coordination under the fourfold core:

```text
空・縁起・二諦 gap・中道
```

## 1. Purpose

The Harmony Function coordinates multiple WORLD models without erasing their differences.

It preserves:

- local WORLD validity,
- membranes,
- gates,
- Qi flow differences,
- Yin-Yang frame differences,
- Wuxing phase differences,
- obstruction visibility,
- governance boundaries.

## 2. Core Formula

The Harmony Function may be expressed as an operational objective:

```text
H_wa = balance(
  non_collapse,
  cross_world_coherence,
  obstruction_visibility,
  compassion_weight,
  governance_safety,
  repairability,
  vitality
)
```

This is not a scalar that overrides all local worlds.

It is a coordination objective used by Mandala operation.

## 5. Harmony Modes

```text
PASS:
  multi-WORLD operation is coherent enough under declared scope

HOLD:
  more observation, support, or boundary declaration is required

REPAIR:
  a WORLD, membrane, transport, or interpretation must be repaired

QUARANTINE:
  a candidate WORLD or transport remains visible but inactive

REJECT:
  operation would collapse, erase, or violate the core

HANDOVER:
  external or higher governance review is required
```

## 6. Non-Collapse Rule

The Harmony Function must not collapse multiple WORLD models into one master WORLD.

It must not allow one WORLD model to become the center.

The center remains:

```text
空・縁起・二諦 gap・中道
```

## 7. Difference Preservation

Disagreement between WORLD models is not automatically failure.

The Harmony Function asks:

```text
Can these differences coexist under declared membranes and gates?
Can obstruction remain visible?
Can repair or re-interpretation preserve safety and meaning?
```

## 8. Relation to Qi, Yin-Yang, and Wuxing

Qi flow may differ by WORLD.

Yin-Yang frame may differ by WORLD.

Wuxing phase may differ by WORLD.

The Harmony Function coordinates these local readings without forcing them into one global reading.

## 10. Guardrails

The Harmony Function must not be used as:

- majority domination,
- forced consensus,
- obstruction erasure,
- authority bypass,
- proof substitute,
- execution license by itself.

Harmony is a governed coordination function, not unrestricted permission.
