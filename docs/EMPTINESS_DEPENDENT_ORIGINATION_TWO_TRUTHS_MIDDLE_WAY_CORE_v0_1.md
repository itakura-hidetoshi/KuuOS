# Emptiness / Dependent Origination / Two Truths / Middle Way Core v0.1

This document defines the KuuOS core boundary for emptiness, dependent origination, the two truths, and the Middle Way.

## Core principle

No artifact, WORLD, provider output, metric, proof stub, audit record, bundle root, or governance decision is self-subsisting.

Every object is a dependent record surface.

## Emptiness

Emptiness means lack of independent self-nature.

In KuuOS, emptiness prevents reification of:

- AI raw output
- WORLD model
- memory record
- belief state
- metric
- audit hash
- WORM receipt
- bundle root
- finality packet

## Dependent origination

Dependent origination means that an output is meaningful only through conditions, lineage, context, boundary, evidence, role, and permitted authority.

KuuOS therefore requires each governed surface to preserve:

- source lineage
- boundary visibility
- role scope
- authority boundary
- uncertainty visibility
- harm visibility

## Two truths

The conventional truth surface contains operational artifacts, records, validators, bundles, and governance states.

The ultimate truth boundary prevents those surfaces from becoming self-existing essence.

KuuOS must preserve both:

- conventional functionality
- ultimate non-reification

## Middle Way

The Middle Way rejects both:

- substantialism: treating artifacts as final self-existing truth
- nihilism: treating artifacts as meaningless or unusable

KuuOS therefore routes artifacts to:

- CANDIDATE
- HOLD
- REPAIR
- REJECT
- QUARANTINE
- OBSERVE

without collapsing into either authority inflation or usefulness denial.
