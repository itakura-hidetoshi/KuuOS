# Claim Level Taxonomy v0.1

## Purpose

KuuOS distinguishes multiple claim levels.

This taxonomy exists to prevent silent escalation between:

- generation
- validation
- reference
- proof
- deployment
- institutional authority

## Claim Levels

| Level | Meaning |
|---|---|
| Candidate | Generated proposal or surface without authority |
| Reference | Informational or citation-level linkage |
| Structural Validation | Validator-confirmed structural consistency |
| Formal Sketch | Incomplete or partially formalized proof-oriented artifact |
| Formal Proof Artifact | Explicit proof artifact under a formal system |
| Canonical Proof Boundary | Declared authoritative theorem repository |
| External Review | Independent external evaluation |
| Institutional Approval | Institutionally recognized authority layer |
| Deployment Authorization | Permission to operate in a live environment |

## Important Separation

KuuOS intentionally separates these levels.

Examples:

- Structural validation is not theorem closure.
- Reference is not proof.
- Candidate generation is not execution authority.
- External review is not institutional approval.

## Repository Orientation

The KuuOS public repository primarily operates in the following levels:

- Candidate
- Reference
- Structural Validation
- Partial Formalization

The repository does not automatically claim:

- Institutional Approval
- Deployment Authorization

## Reviewer Guidance

A reviewer should evaluate whether transitions between claim levels are:

- explicit
- audited
- provenance-preserving
- governance-visible

Silent escalation between claim levels is treated as a governance risk.
